Linux升级内核/boot空间不足解决办法

今天安装更新时，系统提示我/boot空间已满。内核文件默认储存在/boot中，多次升级内核后，内核版本太多，清理没有的内核文件就行。

首先看一下磁盘情况：

输入　df -h

/boot已用100％需要清理了。

查看当前使用的内核版本号。

输入　uname -a

当前内核版本号4.4.0-38

删除旧内核

输入　sudo apt-get remove linux-image- 　接着按两下空格将显示当前储存的所有内核版本，把之前的版本删除就行。刚升级的版本可能不稳定，为了保险保留之前1-2版本备用。
